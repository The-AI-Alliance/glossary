--- layout: redirect --- {% comment %} The template for relative or absolute redirects. In the pages that use it, see for example `docs/example_redirect.markdown`, define either of the following: * redirect_rel_url - for relative redirects relative to this site, that is relative to site.url. * redirect_abs_url - for absolute redirects to a different site. Define one or the other. The code below will look for `redirect_rel_url` first, so if it is defined, `redirect_abs_url` is ignored. The code also appends the location "search" and "hash" from the initial URL. For example, if redirecting form ../foo/bar to ../bip/bop, an input URL like the following will be transformed as shown: /foo/bar/?x=1&y=2#anchor --> /bip/bop/?x=1&y=2#anchor TIP: Make sure you experiment with the definition for these URLs. Be especially careful about relative URLs. Despite the "relative" name, you shouldn't actually use relative constructs like `../`. Use `/foo/bar`, instead, which is the path from the website's root, as defined by site.url in `docs/_config.yml`.
